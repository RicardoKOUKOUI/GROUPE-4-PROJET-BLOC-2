05 — Éléments réutilisables identifiés
Boutons CTA
.c1c a · .e2b a · .gb
3 boutons sur le site avec le même style de base (fond coloré, border-radius, padding, font-weight bold). Ils méritent un fichier buttons.css commun.
Cartes de contenu
.d1–.d4 · .f1–.f3
Les cartes de services et les cartes de témoignages partagent la même structure (flex:1, padding, border-radius, background). Un fichier cards.css peut regrouper ces patterns.
Titres de section
.d0 · .f0 · .g0 · .e2t
Tous les titres de section partagent font-size: 28px, font-weight: bold, color: #1a1a2e (ou white). Potentiellement un style de classe utilitaire dans base.css.
Layouts flex 2 colonnes
.c · .e · .h
3 sections utilisent display:flex avec justify-content:space-between ou gap. Ce pattern de mise en page est répété et pourrait être abstrait dans un futur layout.css.
06 — Problèmes identifiés dans le code du stagiaire
⚠ Problèmes de nommage
  • Classes sans signification sémantique : .b1, .c, .d, .e, .f, .g, .h
  • Impossible de deviner la section depuis le nom de classe
  • Un nouveau développeur doit lire tout le HTML pour comprendre
  • Les commentaires HTML compensent les mauvais noms
⚠ Problèmes d'architecture
  • 1 seul fichier style.css pour 7 sections + styles globaux
  • Les couleurs #1a1a2e et #e94560 sont dupliquées 9+ fois
  • Aucune variable CSS (--color-primary, etc.)
  • Éléments réutilisables (boutons, cartes) non abstraits
✓ Conclusion de la Phase 1
Le site DevStart contient 7 sections bien délimitées, 1 fichier CSS de 230+ lignes couvrant tout sans séparation de responsabilités, et 4 types d'éléments réutilisables non mutualisés. La cartographie est complète et fournit toute la base nécessaire pour concevoir l'architecture modulaire en Phase 2.
